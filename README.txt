Project Structure

This folder contains the following files:

crossword.cpp
The main implementation of the string matching tool.

gentest.cpp
A utility program used to generate test cases for testing the main program.

defaultInput.txt
A default input file for running the program.

defaultOutput.txt
A default output file showing the expected result.

tests
A folder containing generated test cases used for evaluation.

How to Compile and Run

1. Compile the program

Open a terminal and compile the source code using the GNU g++ compiler with the C++17 standard:

g++ crossword.cpp -o crossword -std=c++17

2. Run the program

After compilation, run the program using the following command:

./crossword -a [algorithm] -i [input_file] -o [output_file]

Supported algorithms:

bf
Brute-Force string matching algorithm.

rk
Rabin-Karp string matching algorithm.

kmp
Knuth-Morris-Pratt string matching algorithm.

bm
Boyer-Moore string matching algorithm.

ac
Aho-Corasick algorithm (bonus implementation).

Test Cases Guide

The tests folder contains test cases generated by the gentest.cpp program. Each test case consists of an input file with the extension .input and a corresponding output file with the extension .output.

How to interpret the test case names

The test case filenames are structured to clearly describe the configuration of the dataset.

Example:
random_R_10_C_10_K_10_len_5

R_10
Indicates that the grid contains 10 rows.

C_10
Indicates that the grid contains 10 columns.

K_10
Indicates that the test case contains 10 keywords.

len_5
Indicates that the maximum length of each keyword is 5 characters.
